# Session cockpit owns view status

The mevedel view buffer should present mevedel-owned status and menu entry points instead of proxying gptel's clickable header line. The raw data buffer may keep gptel's normal header, while the view routes mode, model, tools, skills, plugins, and help through the session cockpit. This lets mevedel replace broad gptel transient retargeting advice with a narrow bridge for explicitly opening `gptel-menu` from the paired data buffer.
